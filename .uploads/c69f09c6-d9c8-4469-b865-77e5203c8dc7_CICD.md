# CI/CD 自动化部署流程

## 1. 概述

### 1.1 目标

- 自动化构建、测试和部署
- 支持多环境部署（开发、预发、生产）
- 支持 Web、Windows 桌面端、Android 移动端
- 保障代码质量和部署安全

### 1.2 技术选型

| 组件 | 工具 | 说明 |
|------|------|------|
| CI/CD 平台 | GitHub Actions / Gitea Actions | 开源自托管方案 |
| 容器编排 | Docker Compose | 本地开发与生产 |
| 服务器 | Ubuntu 20.04+ | 目标部署环境 |
| 构建服务器 | 自行搭建 | 运行 GitHub Runner |

### 1.3 部署架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        Git Repository                           │
│                   (GitHub / Gitea / GitLab)                    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼ Push / PR
┌─────────────────────────────────────────────────────────────────┐
│                     CI/CD Pipeline                              │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐           │

```

## 3. GitHub Actions 工作流

### 3.1 持续集成 (ci.yml)

```yaml
# .github/workflows/ci.yml

name: CI

on:
  push:
    branches: [develop, main]
  pull_request:
    branches: [develop, main]

env:
  PHP_VERSION: '8.1'
  NODE_VERSION: '20'

jobs:
  # ============================================
  # 任务1: PHP 代码检查
  # ============================================
  php-lint:
    name: PHP Lint
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup PHP
        uses: shivammathur/setup-php@v2
        with:
          php-version: ${{ env.PHP_VERSION }}
          extensions: swoole, pdo, pdo_mysql, redis, gd, zip, mbstring
          coverage: none

      - name: Install Composer dependencies
        run: |
          composer install --prefer-dist --no-progress

      - name: PHP CodeSniffer
        run: |
          vendor/bin/phpcs --standard=PSR12 app/

      - name: PHPStan Static Analysis
        run: |
          vendor/bin/phpstan analyse app/ --level=5

  # ============================================
  # 任务2: 前端代码检查
  # ============================================
  frontend-lint:
    name: Frontend Lint
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: ESLint
        run: npm run lint

      - name: StyleLint
        run: npm run lint:style

  # ============================================
  # 任务3: 后端单元测试
  # ============================================
  php-test:
    name: PHP Unit Test
    runs-on: ubuntu-latest
    needs: php-lint

    services:
      mysql:
        image: mariadb:10.7
        env:
          MYSQL_ROOT_PASSWORD: root
          MYSQL_DATABASE: test
        ports:
          - 3306:3306
        options: >-
          --health-cmd="mysqladmin ping"
          --health-interval=10s
          --health-timeout=5s
          --health-retries=3

      redis:
        image: redis:7-alpine
        ports:
          - 6379:6379

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup PHP
        uses: shivammathur/setup-php@v2
        with:
          php-version: ${{ env.PHP_VERSION }}
          extensions: swoole, pdo, pdo_mysql, redis, gd, zip, mbstring

      - name: Install Composer dependencies
        run: composer install --prefer-dist --no-progress

      - name: Copy .env
        run: |
          cp .env.example .env
          php artisan key:generate

      - name: Run migrations
        run: php artisan migrate --force

      - name: Run PHPUnit
        run: vendor/bin/phpunit --coverage-text

  # ============================================
  # 任务4: 前端单元测试
  # ============================================
  frontend-test:
    name: Frontend Unit Test
    runs-on: ubuntu-latest
    needs: frontend-lint

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Run Jest
        run: npm run test:unit

  # ============================================
  # 任务5: 构建 Docker 镜像
  # ============================================
  docker-build:
    name: Docker Build
    runs-on: ubuntu-latest
    needs: [php-test, frontend-test]

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Login to Docker Hub
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKER_USERNAME }}
          password: ${{ secrets.DOCKER_TOKEN }}

      - name: Build and push Docker images
        uses: docker/build-push-action@v5
        with:
          context: .
          push: ${{ github.event_name != 'pull_request' }}
          tags: |
            ${{ secrets.DOCKER_USERNAME }}/project:latest
            ${{ secrets.DOCKER_USERNAME }}/project:${{ github.sha }}
          cache-from: type=registry,ref=${{ secrets.DOCKER_USERNAME }}/project:buildcache
          cache-to: type=registry,ref=${{ secrets.DOCKER_USERNAME }}/project:buildcache,mode=max
```

### 3.2 Web 部署 (deploy-web.yml)

```yaml
# .github/workflows/deploy-web.yml

name: Deploy Web

on:
  push:
    branches: [main]
    tags:
      - 'v*'
  workflow_dispatch:
    inputs:
      environment:
        description: '部署环境'
        required: true
        default: 'staging'
        type: choice
        options:
          - staging
          - production

env:
  DEPLOY_USER: deploy
  DEPLOY_HOST: ${{ secrets.DEPLOY_HOST }}
  DEPLOY_KEY: ${{ secrets.DEPLOY_SSH_KEY }}

jobs:
  deploy:
    name: Deploy to ${{ github.event.inputs.environment || 'staging' }}
    runs-on: ubuntu-latest

    environment:
      name: ${{ github.event.inputs.environment || 'staging' }}
      url: ${{ github.event.inputs.environment == 'production' && 'https://your-domain.com' || 'https://staging.your-domain.com' }}

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build frontend
        run: npm run build
        env:
          APP_ENV: production

      - name: Create deployment package
        run: |
          tar -czf deploy.tar.gz \
            --exclude='.git' \
            --exclude='node_modules' \
            --exclude='.github' \
            --exclude='*.log' \
            .

      - name: Deploy to server
        uses: appleboy/scp-action@master
        with:
          host: ${{ secrets.DEPLOY_HOST }}
          username: ${{ env.DEPLOY_USER }}
          key: ${{ env.DEPLOY_KEY }}
          source: deploy.tar.gz
          target: /opt/project/releases/${{ github.run_id }}
          strip_components: 1

      - name: Execute deployment
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.DEPLOY_HOST }}
          username: ${{ env.DEPLOY_USER }}
          key: ${{ env.DEPLOY_KEY }}
          script: |
            # 创建版本目录
            RELEASE_DIR="/opt/project/releases/${{ github.run_id }}"
            CURRENT_DIR="/opt/project/current"
            SHARED_DIR="/opt/project/shared"

            # 创建软链接
            ln -sfn $RELEASE_DIR $CURRENT_DIR

            # 链接共享目录
            ln -sfn $SHARED_DIR/.env $CURRENT_DIR/.env
            ln -sfn $SHARED_DIR/storage $CURRENT_DIR/storage
            ln -sfn $SHARED_DIR/docker/appstore $CURRENT_DIR/docker/appstore

            # 进入目录
            cd $CURRENT_DIR

            # 安装依赖
            composer install --no-dev --optimize-autoloader

            # 运行迁移
            php artisan migrate --force

            # 清理缓存
            php artisan config:cache
            php artisan route:cache
            php artisan view:cache

            # 重启 PHP-FPM
            sudo systemctl reload php-fpm

            # 记录版本
            echo "${{ github.run_id }} - ${{ github.sha }} - ${{ github.event.head_commit.message }}" >> /opt/project/releases.log

            # 清理旧版本（保留最近5个）
            ls -dt /opt/project/releases/*/ | tail -n +6 | xargs -r rm -rf

      - name: Health check
        run: |
          curl -f https://staging.your-domain.com/health || exit 1

      - name: Notify success
        if: success()
        run: |
          echo "部署成功！"
          echo "版本: ${{ github.run_id }}"
          echo "Commit: ${{ github.sha }}"

      - name: Notify failure
        if: failure()
        run: |
          echo "部署失败！"
          echo "请检查服务器日志: /opt/project/releases.log"
```

```yaml
name: Build Windows Desktop

on:
  push:

          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build frontend
        run: npm run build

      - name: Setup PHP

        with:
          php-version: '8.1'

      - name: Install Composer dependencies
        run: composer install --prefer-dist --no-progress

      - name: Build Electron
        run: npm run electron:build -- --win --x64
        env:
          APP_ID: com.yourcompany.project
          PRODUCT_NAME: ProjectName
          APP_VERSION: ${{ env.APP_VERSION }}

      - name: Upload artifacts
        uses: actions/upload-artifact@v4
        with:
          name: windows-installer
          path: |
            electron/dist/*.exe
            electron/dist/*.zip
          retention-days: 30

      - name: Create Release
        if: startsWith(github.ref, 'refs/tags/')
        uses: softprops/action-gh-release@v1
        with:
          files: |
            electron/dist/*.exe
            electron/dist/*.zip
          draft: false
          prerelease: false
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      - name: Upload to Server
        if: startsWith(github.ref, 'refs/tags/')
        run: |
          scp -i ${{ secrets.DEPLOY_SSH_KEY }} \
              -o StrictHostKeyChecking=no \
              electron/dist/*.exe \
              deploy@your-server:/opt/downloads/
```

## 4. 部署脚本

### 4.1 部署脚本 (deploy.sh)

```bash
#!/bin/bash
# scripts/deploy.sh

set -e

# 配置
APP_NAME="project"
APP_DIR="/opt/project"
RELEASE_DIR="$APP_DIR/releases"
SHARED_DIR="$APP_DIR/shared"
CURRENT_DIR="$APP_DIR/current"
KEEP_RELEASES=5

# 颜色
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m'

log_info() {
    echo -e "${GREEN}[INFO]${NC} $1"
}

log_warn() {
    echo -e "${YELLOW}[WARN]${NC} $1"
}

log_error() {
    echo -e "${RED}[ERROR]${NC} $1"
}

# 检查是否为 root 用户
if [ "$EUID" -ne 0 ]; then
    log_error "请使用 sudo 运行此脚本"
    exit 1
fi

# 创建目录
mkdir -p $RELEASE_DIR $SHARED_DIR

# 获取版本号
VERSION=${1:-$(date +%Y%m%d%H%M%S)}
log_info "开始部署版本: $VERSION"

# 切换到发布目录
cd $RELEASE_DIR
mkdir -p $VERSION
cd $VERSION

# 下载和解压代码（这里假设代码已通过 scp 上传）
if [ -f "$RELEASE_DIR/../../../deploy.tar.gz" ]; then
    tar -xzf $RELEASE_DIR/../../../deploy.tar.gz -C .
    rm -f $RELEASE_DIR/../../../deploy.tar.gz
else
    log_error "未找到部署包"
    exit 1
fi

# 链接共享目录
ln -sfn $SHARED_DIR/.env .env
ln -sfn $SHARED_DIR/storage storage
ln -sfn $SHARED_DIR/docker/appstore docker/appstore

# 安装依赖
log_info "安装 PHP 依赖..."
composer install --no-dev --optimize-autoloader --prefer-dist

# 安装前端依赖
log_info "安装前端依赖..."
npm ci --production

# 构建前端
log_info "构建前端..."
npm run build

# 运行数据库迁移
log_info "运行数据库迁移..."
php artisan migrate --force

# 清理缓存
log_info "清理缓存..."
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan event:cache

# 切换软链接
log_info "切换软链接..."
ln -sfn $RELEASE_DIR/$VERSION $CURRENT_DIR

# 重启服务
log_info "重启服务..."
sudo systemctl reload php-fpm
sudo systemctl reload nginx

# 健康检查
log_info "健康检查..."
sleep 5
if curl -sf http://127.0.0.1/health > /dev/null; then
    log_info "健康检查通过"
else
    log_error "健康检查失败"
    exit 1
fi

# 清理旧版本
log_info "清理旧版本（保留最近 $KEEP_RELEASES 个）..."
cd $RELEASE_DIR
ls -dt */ | tail -n +$((KEEP_RELEASES + 1)) | xargs -r rm -rf

# 记录部署日志
echo "[$(date '+%Y-%m-%d %H:%M:%S')] 版本: $VERSION 部署成功" >> $APP_DIR/deploy.log

log_info "部署完成！"
log_info "版本目录: $RELEASE_DIR/$VERSION"
log_info "当前版本: $CURRENT_DIR"
```

### 4.2 备份脚本 (backup.sh)

```bash
#!/bin/bash
# scripts/backup.sh

set -e

# 配置
BACKUP_DIR="/opt/backups"
APP_DIR="/opt/project"
KEEP_DAYS=30

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份文件名
BACKUP_FILE="backup_$(date +%Y%m%d_%H%M%S).tar.gz"

# 备份文件列表
BACKUP_FILES=(
    "$APP_DIR/shared/.env"
    "$APP_DIR/shared/storage"
    "$APP_DIR/shared/docker/appstore"
    "$APP_DIR/shared/public/uploads"
)

# 创建备份
tar -czf $BACKUP_DIR/$BACKUP_FILE "${BACKUP_FILES[@]}"

# 备份数据库
DB_NAME=$(grep DB_DATABASE $APP_DIR/shared/.env | cut -d'=' -f2)
DB_USER=$(grep DB_USERNAME $APP_DIR/shared/.env | cut -d'=' -f2)
DB_PASS=$(grep DB_PASSWORD $APP_DIR/shared/.env | cut -d'=' -f2)

mysqldump -u$DB_USER -p$DB_PASS $DB_NAME | gzip > $BACKUP_DIR/db_$(date +%Y%m%d_%H%M%S).sql.gz

# 清理旧备份
find $BACKUP_DIR -name "backup_*.tar.gz" -mtime +$KEEP_DAYS -delete
find $BACKUP_DIR -name "db_*.sql.gz" -mtime +$KEEP_DAYS -delete

echo "备份完成: $BACKUP_DIR/$BACKUP_FILE"
```

### 4.3 健康检查脚本 (healthcheck.sh)

```bash
#!/bin/bash
# scripts/healthcheck.sh

# 检查 Nginx
if ! pgrep -x nginx > /dev/null; then
    echo "Nginx 未运行"
    exit 1
fi

# 检查 PHP-FPM
if ! pgrep -x php-fpm > /dev/null; then
    echo "PHP-FPM 未运行"
    exit 1
fi

# 检查 MySQL
if ! mysqladmin ping -h127.0.0.1 > /dev/null 2>&1; then
    echo "MySQL 未运行"
    exit 1
fi

# 检查 Redis
if ! redis-cli ping > /dev/null 2>&1; then
    echo "Redis 未运行"
    exit 1
fi

# 检查应用健康端点
HTTP_CODE=$(curl -sf -o /dev/null -w "%{http_code}" http://127.0.0.1/health)
if [ "$HTTP_CODE" != "200" ]; then
    echo "应用健康检查失败 (HTTP $HTTP_CODE)"
    exit 1
fi

echo "OK"
exit 0
```

## 5. Docker 配置

### 5.1 生产环境 Docker Compose

```yaml
# docker/docker-compose.yml

services:
  # PHP 应用
  php:
    container_name: ${APP_NAME}-php
    build:
      context: ../
      dockerfile: docker/php/Dockerfile
    shm_size: 2G
    volumes:
      - ../:/var/www
      - app_storage:/var/www/storage
      - app_uploads:/var/www/public/uploads
    environment:
      - APP_ENV=production
      - APP_DEBUG=false
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy
    restart: unless-stopped
    networks:
      - app-network

  # Nginx
  nginx:
    container_name: ${APP_NAME}-nginx
    image: nginx:alpine
    ports:
      - "${APP_PORT}:80"
      - "${APP_SSL_PORT:-0}:443"
    volumes:
      - ../:/var/www
      - ./nginx/default.conf:/etc/nginx/conf.d/default.conf
      - app_storage:/var/www/storage
      - app_uploads:/var/www/public/uploads
      - ./logs/nginx:/var/log/nginx
    depends_on:
      - php
    restart: unless-stopped
    networks:
      - app-network

  # MySQL
  mysql:
    container_name: ${APP_NAME}-mysql
    image: mariadb:10.7
    volumes:
      - mysql_data:/var/lib/mysql
      - ./mysql/my.cnf:/etc/mysql/conf.d/my.cnf
    environment:
      - MYSQL_ROOT_PASSWORD=${DB_ROOT_PASSWORD}
      - MYSQL_DATABASE=${DB_DATABASE}
      - MYSQL_USER=${DB_USERNAME}
      - MYSQL_PASSWORD=${DB_PASSWORD}
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5
    restart: unless-stopped
    networks:
      - app-network

  # Redis
  redis:
    container_name: ${APP_NAME}-redis
```

### 8.2 监控脚本

```bash
#!/bin/bash
# scripts/monitor.sh

# 检查服务状态
check_service() {
    local service=$1
    local url=$2

    if curl -sf "$url" > /dev/null 2>&1; then
        echo "[OK] $service"
    else
        echo "[FAIL] $service"
        # 可以添加告警通知
    fi
}

# 定期检查
while true; do
    check_service "Web" "http://127.0.0.1/health"

    # 检查磁盘空间
    DISK_USAGE=$(df / | tail -1 | awk '{print $5}' | sed 's/%//')
    if [ "$DISK_USAGE" -gt 90 ]; then
        echo "[WARN] 磁盘使用率: ${DISK_USAGE}%"
    fi

    # 检查内存
    MEM_USAGE=$(free | grep Mem | awk '{printf("%.0f"), $3/$2 * 100}')
    if [ "$MEM_USAGE" -gt 90 ]; then
        echo "[WARN] 内存使用率: ${MEM_USAGE}%"
    fi

    sleep 60
done
```

## 9. 回滚流程

### 9.1 自动回滚脚本

```bash
#!/bin/bash
# scripts/rollback.sh

set -e

APP_DIR="/opt/project"
CURRENT_DIR="$APP_DIR/current"
RELEASES_DIR="$APP_DIR/releases"

# 获取上一个版本
PREV_VERSION=$(ls -dt $RELEASES_DIR/*/ | sed -n '2p' | xargs basename)

if [ -z "$PREV_VERSION" ]; then
    echo "没有可回滚的版本"
    exit 1
fi

echo "开始回滚到版本: $PREV_VERSION"

# 切换软链接
ln -sfn $RELEASES_DIR/$PREV_VERSION $CURRENT_DIR

# 清理缓存
cd $CURRENT_DIR
php artisan config:cache
php artisan route:cache

# 重启服务
sudo systemctl reload php-fpm
sudo systemctl reload nginx

echo "回滚完成！"
```

## 10. 工作流程总结

### 10.1 开发流程

```
1. 从 develop 创建功能分支
   git checkout -b feature/user-auth

2. 开发并提交代码
   git commit -m "feat(user): 添加用户认证功能"

3. 推送到远程
   git push origin feature/user-auth

4. 创建 Pull Request
   -> GitHub 创建 PR

5. CI 自动运行
   -> 代码检查
   -> 单元测试
   -> 构建测试

6. 合并到 develop
   -> 自动部署到预发环境

7. 准备发布
   -> 从 develop 创建 release 分支
   -> 或打 tag: git tag v1.0.0

8. 合并到 main
   -> 自动部署到生产环境
   -> 自动构建 Windows APK
   -> 自动构建 Android APK
```

### 10.2 部署触发条件

| 分支 | 触发条件 | 部署环境 |
|------|----------|----------|
| feature/* | PR 创建 | - |
| develop | Push | 预发环境 |
| main | Push / Tag | 生产环境 |
| v* | Tag | 生产环境 + 构建客户端 |
